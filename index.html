> Qu'est ce que Git ?

Git est un logiciel de contrôle de version décentralisé utilisé pour le développement de projet, souvent logiciel.

C'est un logiciel libre, par l'auteur du noyau Linux: Linus Torvalds.

Sa principale fonction est de gérer l'évolution du contenu d'une arborescence de versions de logiciel. Il stocke sur le disque local les différentes version d'un fichier.

Ce système de version décentralisé permet un développement facilité des logiciels libres. En effet, il suffit de cloner (fork) le projet libre en cours et ainsi obtenir une version locale du projet (ainsi que toutes les anciennes versions). Les modifications ensuite effectuées localement n'ont pas d'effet immédiat sur le projet principal. Il suffit ensuite d'effectuer une pull request au dossier principal pour soumettre ces modifications à la tête du projet. Ce procédé évite donc énormément d'écueils du système de sauvegarde projet centralisé.

Git permet donc un contrôle de version d'un projet via un système de Commit:

Le Commit est un objet contenant toutes les informations concernant les mises à jour des fichiers modifiés depuis le dernier objet commit. C'est l'unité de sauvegarde de Git. Chaque version du projet a donc son commit. Les commit sont classés par date (et se voient attribués un numéro de révision unique) et sont une suite incrémentale des changements appliqués sur le projet (ajout, modification ,suppression de fichiers). Tout est gardé et récupérable qu'elle que soient les modifications apportées entre temps.

Git possède également un système de branche de projet. Chaque branche d'un projet aura son propre historique de commits. Les branches sont des versions divergentes du même projet, par exemple pour tester des prototypes d'algorithme sur plusieurs versions avant de merge ces branches en une version commune.

Git possède deux interfaces:

  • Une interface web permettant de visualiser le dossier central du projet.

  • Une interface console après l'installation du module git. C'est via cette interface que l'on effectue les fonctions de commit ou de branche.

Ce site a pour fonction de répertorié ces principales commandes console de git et de donner une explication succinte de leur fonctionnement.
