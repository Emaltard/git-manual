git revert

La commande permet de rétablir des commits déjà éxistant. Elle sert donc à "remonter le temps" et annuler l'effet de certains commit précédemment fait En réalité, la commande ne supprime pas le commit choisit mais créé un commit automatique sans les modifications du commit choisit, à ne pas confondre avec git reset. Dans l'exemple, on fera un retour en arrière de 1 commit.

Syntaxe :

git revert

Affichage

git revert HEAD
git revert HEAD
[choisir le nom du commit]

git log
commit ...
Author: x
Date: YY
Le commit revert

git revert -e

Cette commande permet d'éditer le nouveau commit qui "supprimera" le commit choisit. La fonction -e ou --edit est l'option par defaut dans un terminal.

Syntaxe

git revert -e

Affichage

git revert -e HEAD
[choisir le nom du commit]

git log
commit ...
Author: x
Date: YY
Le commit revert

git revert -n

Normalement la commande git revert crée automatiquement des commit dont les messages renseignent les commit annulés. Ici, on annule effectivement les commit désirés dans le répertoire git et l'index, mais on ne crée pas de comit.De plus, l'index n'as pas à correspondre avec le commit HEAD (utile notamment lorsque l'on a effecué un git revert de plusieurs commit touchant a l'index d'afilée).

Syntaxe :

git revert -n
