Les différents moyens d'identifications des commits

SHA

Un Sha est une série de 40 chiffres héxadécimal généré à l'aide d'un algorithme dont le nom est SHA-1(pour Secure Hash Algorithm). Vous pouvez connaitre la SHA d'un commit en utilisant la commande log.

Exemple d'affichage d'un SHA

Exemple d'un log d'un commit :

commit 283081424dce5ea4a237bceedeb846d914ea8d2f
Author: Utilisateur
Date: Thu Mar 12 08:42:17 2015 +0100
Message de commit
Commentaire sur le commit

Le SHA est unique à un commit, il est formé grâce aux méta-données du commit, c'est-à-dire sa date, les informations de l'auteur, ainsi que le message de commit. Pour utilisez un SHA, vous n'êtes pas obliger de l'écrire entièrement, seule les quelques premiers chiffres devraient suffir si un autre commit ne les partagent pas dans votre projet.

Il existe d'autres façons de référencé des commits.

Le nom d'une branche

Une branche est une version divergente du projet, comme pour la branche master sur laquelle vous travaillez généralement, le nom d'une branche référence sur le dernier commit de cette branche (ainsi une branche peut référencé un commit du passé de la branche master). Pour plus d'informations sur les branches vous pouvez aller voir la commande branch.

Exemple d'affichage des branches

Exemple d'un git branch -v :

* gh-pages 2830814 Merge pull request #74 from Iwantaponey/gh-pages

master 2374582 Mise à jour de commandes.html

Ici, nous avons deux branches pointant sur des commits différents. Si l'on utilise l'identificateur gh-pages l'on parlera alors du commit Merge pull request #74 from Iwantaponey/gh-pages alors que master ferait référence au commit Mise à jour de commandes.html

Pour plus d'informations sur les branches, n'hésitez pas à aller voir la page de la commande branch

D'autres moyens de référencé les commits

  • Les tag, vous pouvez en apprendre plus sur les tags ici.

  • L'utilisation d'un sélécteur de parent : ^ ou ~ qui fait référence au commit précédent. Par exemple, HEAD^ fait référence à l'avant dernier commit de la branche courante.
  • Une référence à partir d'une date à l'aide de @ suivis de la date du commit.
  • Une référence sur le chemin du commit à l'aide de :.
