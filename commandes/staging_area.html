Les différents répertoires dans git

  • Accueil

La working directory

La working directory représente le dossier dans le quel vous travaillez actuellement, avec les versions de la branche actuelle de votre travail. Il vous est propre et est mis à jour avec les versions du projet éloigné à l'aide des commandes push et pull.

La staging area

C'est le fichier qui contient les fichiers et modifications apportés qui sont en attente d'un commit.
C'est-à-dire qui on été ajouté à l'aide de la commande add.
Ou modifié avec d'autres commandes tel que git mv et git rm.
Après un simple calcul de différences, git peut ainsi savoir si les dernières modifications ont été commités.

Le git repository

Le git repository (ou history tree) est le fichier .git habituellement caché, il contient l'historique des commits du projet. Il garde en mémoire tout les changements qui ont été commités. C'est ce qui est cloné quand on clone la version d'un "serveur".

Répertoire nu, ou bare repository

Un répertoire nu est le type de répertoire que l'on retrouve sur un serveur d'échange. (Par exemple sur github).
On peut créer un répertoire nu à l'aide de la commande git init --bare
ou cloner un dépôt git déjà existant à l'aide de git clone --bare url.
